Remnawave-Scripts — это кроссплатформенный Bash-скрипт с интерактивным меню, поддержкой двух языков (RU/EN), генерацией случайных ID, поиском стран с эмодзи-флагами и управлением портами.

Также включает систему самообновления с GitHub и позволяет удалить скрипт (rw-scripts) прямо из меню.

---

## 📌 Возможности

- **Генерация случайного shorts_id** — создание уникальных идентификаторов
- **Поиск стран** — поиск на русском и английском, поддерживаются частичные совпадения
- **Показать эмодзи-флаг** и английское название страны
- **💾 Мониторинг памяти** ⭐ v1.1.0
  - Отображение свободной и занятой оперативной памяти
  - Кроссплатформенная поддержка (Linux, macOS)
- **📈 Интерактивный монитор процессов (htop)** ⭐ v1.1.0
  - Автоматическая установка htop при необходимости
  - Мониторинг процессов, CPU и памяти в реальном времени
- **🔒 Управление портами** ⭐ v1.2.0
  - 🔓 Открытие портов через firewall (UFW/FirewallD/iptables)
  - 🔒 Закрытие портов с удалением из базы
  - ✏️ Редактирование описаний портов
  - 📋 Красивый список всех открытых портов
  - 🛡️ Просмотр статуса firewall
  - 💾 Сохранение данных портов между сеансами
- **📦 Обновление системных пакетов** ⭐ НОВОЕ в v1.2.3
  - Выполнение `apt update && apt upgrade -y` из меню
  - Подтверждение перед запуском
  - Поддержка apt и apt-get
- **Информация о системе** — отображение данных системы через neofetch
- **Переключение языка**: Русский / English
- **Интерактивное CLI-меню**
- **Система обновления через GitHub**
- **Возможность удаления** из системы

## 🚀 Установка

Установка одной командой:

```bash
bash <(curl -s https://raw.githubusercontent.com/detective-noir-dev/Remnawave-Scripts/main/install.sh)
```

Запуск глобально:

```bash
rw-scripts
```

---

## ⚙️ Удаление

Если вам больше не нужен **Remnawave Scripts**, вы можете удалить его:

```bash
rw-scripts
# Выберите пункт 4 → 3 (Удалить rw-scripts)
```

Или напрямую:

```bash
~/.local/share/remnawave/uninstall.sh
```

## 📖 Использование

### 📁 Группы меню

| # | Группа | Содержимое |
|---|--------|------------|
| 1 | 🆔 Short ID & Флаги стран | Генерация shorts_id, Получение emoji-флага страны |
| 2 | 📊 Монитор ресурсов | Свободная память, htop, Системная информация (neofetch) |
| 3 | 🔐 Сеть и порты | Подменю управления портами |
| 4 | ⚙️ Обслуживание | Проверка обновлений, **Обновление системных пакетов** ⭐, Удаление |

---

### Примеры использования

#### 💾 Мониторинг памяти
Отображает информацию о:
- Общей памяти
- Используемой памяти
- Свободной памяти
- Кэшированной памяти

#### 📈 Монитор процессов
- Интерактивный интерфейс htop
- Сортировка по CPU/памяти
- Управление процессами
- Цветное отображение данных

#### 🔒 Управление портами (группа 3)

**Подменю управления портами:**

```
╔════════════════════════════════════════════╗
║      🔒 Управление портами / Ports        ║
╚════════════════════════════════════════════╝

1) 🔓 Открыть порт / Open port
2) 🔒 Закрыть порт / Close port
3) ✏️  Редактировать описание / Edit description
4) 📋 Список портов / List ports
5) 🛡️  Статус firewall / Firewall status
0) ⬅️  Назад / Back
```

**Возможности:**

1. **Открыть порт**
   - Выбор номера порта (1-65535)
   - Выбор протокола (TCP, UDP или оба)
   - Добавление описания (например: "SSH server", "Web server")
   - Автоматическое добавление правила в firewall

2. **Закрыть порт**
   - Просмотр списка открытых портов
   - Выбор порта для закрытия
   - Автоматическое удаление из firewall и базы данных

3. **Редактировать описание**
   - Изменение описания существующего порта
   - Полезно для документирования назначения портов

4. **Список портов**
   ```
   ┌─────────┬───────────┬──────────────────────────────────────┐
   │ Port    │ Protocol  │ Description                          │
   ├─────────┼───────────┼──────────────────────────────────────┤
   │ 22      │ tcp       │ SSH server                           │
   │ 80      │ tcp       │ Web server (HTTP)                    │
   │ 443     │ tcp       │ Web server (HTTPS)                   │
   │ 3306    │ tcp       │ MySQL Database                       │
   └─────────┴───────────┴──────────────────────────────────────┘
   📊 Всего открытых портов: 4
   ```

5. **Статус firewall**
   - Показывает тип используемого firewall (UFW/FirewallD/iptables)
   - Выводит активные правила
   - Отображает общий статус защиты

**Поддерживаемые firewall:**
- ✅ **UFW** (Ubuntu/Debian)
- ✅ **FirewallD** (RHEL/CentOS/Fedora)
- ✅ **iptables** (универсальный для Linux)

#### 📦 Обновление системных пакетов (группа 4 → пункт 2) ⭐ НОВОЕ в v1.2.3

**Подменю обслуживания:**

```
╔════════════════════════════════════════════╗
║         ⚙️  Обслуживание                  ║
╚════════════════════════════════════════════╝

1) 🔄 Проверить обновления
2) 📦 Обновить системные пакеты    ⭐ НОВОЕ
3) 🗑️  Удалить rw-scripts
0) ⬅️  Назад
```

**Функция обновления пакетов:**
- Выполняет `apt update && apt upgrade -y`
- Запрашивает подтверждение перед запуском
- Автоматически определяет `apt` или `apt-get`
- Показывает прогресс обновления в реальном времени
- Выводит сообщение об успешном завершении

**Пример использования:**
```
📦 Обновить системные пакеты? Это может занять время. (y/n)
> y

📦 Обновление списка пакетов...
Hit:1 http://archive.ubuntu.com/ubuntu jammy InRelease
...

⬆️  Установка обновлений...
Reading package lists... Done
...

✅ Все пакеты обновлены!
```

**Примечание:** Эта функция доступна только для систем на базе Debian/Ubuntu с пакетным менеджером apt.

## 📦 Системные требования

- **ОС**: Linux, macOS, BSD, Windows WSL
- **Зависимости** (устанавливаются автоматически при необходимости):
  - `bash` — основной интерпретатор
  - `curl` — для загрузки обновлений
  - `xxd` — для генерации ID (автоустановка)
  - `neofetch` — для информации о системе (опционально)
  - `htop` — для монитора процессов (опционально)
  - `jq` — для работы с JSON в управлении портами (автоустановка)
  - `ufw` / `firewalld` / `iptables` — firewall для управления портами
  - `apt` / `apt-get` — для обновления системных пакетов (Debian/Ubuntu)

## 🔐 Безопасность

При использовании скрипта:

**Управление портами:**
- ⚠️ Требуются права sudo для работы с firewall
- 🛡️ Все изменения применяются напрямую к системному firewall
- 💾 Данные портов хранятся локально в вашей домашней директории
- 🔒 Рекомендуется открывать только необходимые порты

**Обновление системных пакетов:**
- ⚠️ Требуются права sudo для установки пакетов
- 📦 Выполняется полное обновление системы
- ⏱️ Процесс может занять значительное время
- 🔄 Рекомендуется выполнять регулярно для безопасности системы
